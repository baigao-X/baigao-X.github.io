## 概念

- **Pod**: Kubernetes 中最小的部署單元。一個 Pod 包含一個或多個緊密關聯的容器，它們共享網絡和存儲。你可以把它想像成一個獨立的“虛擬主機”。
- **Deployment**: 這是我們主要使用的對象。它是一個控制器，用來管理 Pod 的生命週期。它確保你指定數量的 Pod 副本（Replicas）一直處於運行狀態。當你需要更新應用或回滾時，也是通過操作 Deployment 來完成的。
- **Service**: Pod 的生命週期是短暫的，它們可能會被銷毀和重建，每次重建 IP 地址都可能改變。Service 提供了一個穩定的網絡端點（固定的 IP 地址和 DNS 名稱），用於訪問一組功能相同的 Pod。它充當了 Pod 前面的“負載均衡器”。
- **Ingress**: 调度nigix的负载均衡器 
- **Namespace**: 用於在一個物理集群中創建多個“虛擬集群”。它可以將資源進行邏輯上的隔離，適用於多團隊、多環境（開發、測試、生產）的場景。

### Namespace 

**名字空间（Namespace）** 提供一种机制，将同一集群中的资源划分为相互隔离的组。 同一名字空间内的资源名称要唯一，但跨名字空间时没有这个要求。
