# 延时专题<no value>

**音频在设备端上的延时：**
- **音频采集延时**：采集后的音频首先会经过声卡进行信号转换，声卡本身会产生延时，比如 M-Audio 声卡设备延迟 1ms，[艾肯声卡](https://zhida.zhihu.com/search?content_id=6591992&content_type=Article&match_order=1&q=%E8%89%BE%E8%82%AF%E5%A3%B0%E5%8D%A1&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NDcyODgyNTQsInEiOiLoib7ogq_lo7DljaEiLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjo2NTkxOTkyLCJjb250ZW50X3R5cGUiOiJBcnRpY2xlIiwibWF0Y2hfb3JkZXIiOjEsInpkX3Rva2VuIjpudWxsfQ.fJooL-vw5q4NfrHWEv1pNIee3nvvdi0B_H9iiI-mHI4&zhida_source=entity)设备延迟约为 37ms；
- **编解码延时**：随后音频进入前处理、编码的阶段，如果采用 OPUS [标准编码](https://zhida.zhihu.com/search?content_id=6591992&content_type=Article&match_order=1&q=%E6%A0%87%E5%87%86%E7%BC%96%E7%A0%81&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NDcyODgyNTQsInEiOiLmoIflh4bnvJbnoIEiLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjo2NTkxOTkyLCJjb250ZW50X3R5cGUiOiJBcnRpY2xlIiwibWF0Y2hfb3JkZXIiOjEsInpkX3Rva2VuIjpudWxsfQ.6OzNeHT6J_ZzCInb0Njias8IcKGf52tKMoNIbhFbFwE&zhida_source=entity)，最低算法延时大约需要 2.5~60ms；
- **音频播放延时**：这部分延时与播放端硬件性能相关。
- **音频处理延时**：前后处理，包括 AEC，ANS，AGC 等前后处理算法都会带来算法延时，通常这里的延时就是[滤波器阶数](https://zhida.zhihu.com/search?content_id=6591992&content_type=Article&match_order=1&q=%E6%BB%A4%E6%B3%A2%E5%99%A8%E9%98%B6%E6%95%B0&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NDcyODgyNTQsInEiOiLmu6Tms6LlmajpmLbmlbAiLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjo2NTkxOTkyLCJjb250ZW50X3R5cGUiOiJBcnRpY2xlIiwibWF0Y2hfb3JkZXIiOjEsInpkX3Rva2VuIjpudWxsfQ.OaEh4LaQE1Q9JbUnJjbIRvvjmRqLnYQx8dipar0QPAk&zhida_source=entity)。在 10ms 以内。
- **端网络延时**：这部分延时主要出现在解码之前的 jitter buffer 内，如果在抗丢包处理中，增加了重传算法和前向[纠错算法](https://zhida.zhihu.com/search?content_id=6591992&content_type=Article&match_order=1&q=%E7%BA%A0%E9%94%99%E7%AE%97%E6%B3%95&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NDcyODgyNTQsInEiOiLnuqDplJnnrpfms5UiLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjo2NTkxOTkyLCJjb250ZW50X3R5cGUiOiJBcnRpY2xlIiwibWF0Y2hfb3JkZXIiOjEsInpkX3Rva2VuIjpudWxsfQ.QY11hACtHNAVpE3ndRgNYEnZJPY4x-PmIrxiAvKkg7E&zhida_source=entity)，这里的延时一般在 20ms 到 200ms 左右。但是受到 [jitter buffer](https://zhida.zhihu.com/search?content_id=6591992&content_type=Article&match_order=2&q=jitter+buffer&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NDcyODgyNTQsInEiOiJqaXR0ZXIgYnVmZmVyIiwiemhpZGFfc291cmNlIjoiZW50aXR5IiwiY29udGVudF9pZCI6NjU5MTk5MiwiY29udGVudF90eXBlIjoiQXJ0aWNsZSIsIm1hdGNoX29yZGVyIjoyLCJ6ZF90b2tlbiI6bnVsbH0.cBKBNIsKIk_fdstOxxUd156nujRmT9mi80k8JcGMU_g&zhida_source=entity) 影响，可能会更高。

**视频在设备端上的延时**：

- **采集延时**：采集时会遇到成像延迟，主要由 CCD 相关硬件产生，市面上较好的 CCD 一秒可达 50 帧，成像延时约为 20ms，如果是一秒 20~25 帧的 CCD，会产生 40~50ms 的延时；
		CCD 爆光延时: 2.5微妙，CMOS 速度更快
- **编解码延时**：以 H.264 为例，它包含 I、P、B 三种帧（下文会详细分析），如果是每秒 30 帧相连帧，且不包括 B 帧（由于 B 帧的解码依赖前后视频帧会增加延迟），采集的一帧数据可能直接进入[编码器](https://zhida.zhihu.com/search?content_id=6591992&content_type=Article&match_order=1&q=%E7%BC%96%E7%A0%81%E5%99%A8&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NDcyODgyNTQsInEiOiLnvJbnoIHlmagiLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjo2NTkxOTkyLCJjb250ZW50X3R5cGUiOiJBcnRpY2xlIiwibWF0Y2hfb3JkZXIiOjEsInpkX3Rva2VuIjpudWxsfQ.p0sxVqBsEVFKodyuH--BOUBgTer2YIIoncEEv19M27I&zhida_source=entity)，没有 B 帧时，编码的帧延时可以忽略不计，但如果有 B 帧，会带来算法延时。
- **视频渲染延时**：一般情况下渲染延时非常小，但是它也会受到[系统性能](https://zhida.zhihu.com/search?content_id=6591992&content_type=Article&match_order=1&q=%E7%B3%BB%E7%BB%9F%E6%80%A7%E8%83%BD&zd_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJ6aGlkYV9zZXJ2ZXIiLCJleHAiOjE3NDcyODgyNTQsInEiOiLns7vnu5_mgKfog70iLCJ6aGlkYV9zb3VyY2UiOiJlbnRpdHkiLCJjb250ZW50X2lkIjo2NTkxOTkyLCJjb250ZW50X3R5cGUiOiJBcnRpY2xlIiwibWF0Y2hfb3JkZXIiOjEsInpkX3Rva2VuIjpudWxsfQ.ZccONvurV0-95alh86a-kIjBuy-uPZjsrmLwyz1pBx8&zhida_source=entity)、音画同步的影响而增大。
- **端网络延时**：与音频一样，视频也会遇到端网络延时。
