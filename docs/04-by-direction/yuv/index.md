# Yuv<no value>

#### **1. 采样方式（色度下采样）**
- **YUV444**：
    - **特点**：每个像素独立采样Y、U、V分量，无信息损失。
    - **带宽占用**：与RGB相同（每像素3字节）。
    - **适用场景**：专业视频编辑、医疗成像等对色彩精度要求高的领域。
- **YUV422**：
    - **特点**：水平方向每2个Y共用1组UV，垂直方向全采样。
    - **带宽占用**：每像素2字节（比RGB节省33%）。

- **YUV420**：
	- **特点**：每2×2像素的4个Y共用1组UV，水平和垂直方向均下采样。
	- **带宽占用**：每像素1.5字节（比RGB节省50%）。
	- **优势**：广泛用于流媒体（H.264/AVC）、视频会议等带宽敏感场景


#### **2. 存储结构（内存排列方式）**

##### **(1) 平面格式（Planar）**
- **I420（YU12）**：
	- 存储顺序：`YYYYYYYY → UUUU → VVVV`
- **YV12**：
	- 存储顺序：`YYYYYYYY → VVVV → UUUU`（与I420的UV顺序相反）

##### **(2) 半平面格式（Semi-Planar）**
	Y独立存储，UV交错存储在同一个平面：
- **NV12**：
    - 存储顺序：`YYYYYYYY → UVUVUV...`
    - 典型应用：iOS系统、微软DirectX 。
- **NV21**：
    - 存储顺序：`YYYYYYYY → VUVUVU...`
    - 典型应用：Android系统摄像头输出
##### **(3) 打包格式（Packed）**

YUV分量按像素交错存储在同一数组：

- **YUYV（YUV422）**：
    - 结构：`(Y00, U00, Y01, V00), (Y02, U01, Y03, V01), ...`
    - 每4字节存储2像素 。
- **UYVY**：
    - 结构：`(U00, Y00, V00, Y01), (U01, Y02, V01, Y03), ...`（UV在前）
