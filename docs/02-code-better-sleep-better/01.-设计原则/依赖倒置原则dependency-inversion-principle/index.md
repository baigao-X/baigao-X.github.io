# 依赖倒置原则(Dependency Inversion Principle)<no value>

##  1. Dependency injection 依赖注入
	对象和函数接受所依赖的其他对象或函数。达到分离构造对象和使用对象的关注点。
	1. 将配置项外部化到配置文件中

### 1.1 优点：
1. 减少类与依赖项之间的耦合
2. 便于单元测试

### 1.2 缺点：
	1. 代码难以追踪，因为他的行为与构造是分离的

### 1.3 关键4个角色

#### 1.3.1 服务：
	指包含有用功能能够的类
#### 1.3.2. 客户端
	使用服务的类
#### 1.3.3. 接口
#### 1.3.4. 注入器
	也可以称为组装器、容器、提供者、工厂，向客户端引入服务
