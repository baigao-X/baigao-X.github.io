## 3. 二叉树
- 叶子节点： 左右成员都为null 
- 二叉树节点的深度：指从根节点到该节点的最长简单路径边的条数或者节点数（取决于深度从0开始还是从1开始）
- 二叉树节点的高度：指从该节点到叶子节点的最长简单路径边的条数或者节点数（取决于高度从0开始还是从1开始）
- 最大深度： 二叉树的深度为根节点到最远叶子节点的最长路径上的节点数。
- 最小深度：从根节点到最近叶子节点的最短路径上的节点数量。
- 前/中/后 序遍历
- 对称二叉树
- 平衡二叉树： 每个节点的左右子数高度差不大于1
- 完全二叉树
	- 堆： 实际结构是完全二叉树
		 - 节点下标为i,则左子节点下标为2i+1,右子节点为2i+2
		-  大根堆/大顶堆： 完全二叉树树的一种，每个父节点都比其左右子节点的值大
			 - 基本操作：
				 - 下沉(Sink)：根节点破坏了堆序性: 从根节点开始与最大的子节点交换，持续比较，交换直到满足堆序性。 
					 - 复杂度为O(log N)
				 - 上浮(Swim)：最后一个节点破坏了堆序性:与父节点比较交换，持续直到满足堆序性
					 - 复杂度为O(log N)
			- 建堆：
				- 自顶向下建立：可以理解为持续的**插入**。时间复杂度为O(n log n)
				- 自下而上建立：先初始化堆，从倒数第二排的节点开始做**下沉**操作。直到根节点。 时间复杂度为O(n)
			- 插入： 将新元素加入到最后一个节点，然后进行**上浮**操作，时间复杂度 O(log n)
			- 弹出堆顶元素： 移除根节点元素，然后将最后一个节点移动到根元素，再进行**下沉**操作，时间复杂度O(log n)
		- 小根堆/小顶堆
			- 优先队列是一种小根堆
- 满二叉树
- 最大二叉树 : 根节点是最大值
-  二叉搜索树（BST）：
	1.  若它的左子树不空，则左子树上所有结点的值均小于它的根结点的值；
	2. 若它的右子树不空，则右子树上所有结点的值均大于它的根结点的值；
	3. 它的左、右子树也分别为二叉搜索树
- 平衡二叉搜索树  AVL 树
	- 插入、删除、查找:严格O(logN)
- 红黑树(自平衡的二叉搜索树)
    红黑树在二叉搜索树的基础上增加颜色属性（红/黑），满足以下5条性质
    - ​**​根节点为黑色​**​。
    - ​**​叶子节点（NIL空节点）为黑色​**​。
    - ​**​红色节点的子节点必须是黑色​**​（无连续红节点）。
    - ​**​任意节点到叶子节点的路径包含相同数量的黑色节点​**​（黑高一致）。
    - ​**​节点非红即黑​**​。
    最长路径不超过最短路径的2倍（最短路径全黑，最长路径红黑交替）
	- 插入、删除、查找:平均O(logN)
	- 相对AVL树，修改数据消耗更低
- 累加树
	每个节点 `node` 的新值等于原树中大于或等于 `node.val` 的值之和。

# 算法思路	

## 1. 排序

### 指标
- 时间复杂度
- 空间复杂度
- 稳定性:已经有序的对象，再排序，顺序是否会变
- 原地排序：是否需要额外空间
### 分类

![[v2-b92f60067804733195466a192e0b3603_720w.jpg]]
#### 选择排序
步骤：N从0到size-1, 每次从数组[N->size-1]中寻找最N小的元素，并序号N的数值位置对调。

--> 解决不稳定问题：需要将**对调交换**操作修改为 再次执行for循环进行**按照顺序移动存放**
#### 冒泡排序
**选择排序**的有优化(合并查找第N小的数和按照顺序存放步骤)，解决不稳定问题
可以通过是否进行过交换，来判断是否已经有序来提前终止
步骤：N从0到size-1,每次数组从[size-1->N]开始遍历,每次当右侧数值小于左侧时，对调两个数。这样每次循环可以得到最N小的数值。

#### 插入排序
**选择排序**的优化（合并查找第N小的数和按照顺序存放步骤），解决不稳定问题。
对于本身有序的对象，时间消耗更少。如果本身逆序，消耗会最大。
步骤:N从0到size-1,每次将前N个数据视为一个数组，使有序。每次第N个数，在数组[0->N-1]中插入对应的序号位置。

#### 希尔排序
对**插入排序**的优化。通过分组逐步提升数组的有序度，来提升插入排序的效率。
步骤：设定一个分组间隔h。 将0,0+h,0+2h... 视为一组，1,1+h,1+2h...视为一组进行插入排序
       将h=h/2,再次分组排序
	  直到h=1，再次排序。
时间复杂度: O(n^k) (1.3<=k<=2) .平均时间复杂度  O(n log n).

#### 快速排序
思路： 二叉树的**前序遍历**
步骤：设i=0，j=size-1. i++,直到i的数值大于j的数值，对调。改为j--。依次类推。
	  当i=j时，确定第i个数值是数组中第i大的数值(左边的数据都比它小，右侧的数据都比它大）
	  以上动作等同于，以数组最后一个值为基准，使(左边的数据都比它小，右侧的数据都比它大）。
      然后递归将左右两个数组依次按照上述思路排序。
      N每次选取当前待处理数组的(size-1)/2。
基准数的选择会影响时间复杂度
	  
#### 归并排序
二叉树的**后序遍历**
思路:由合并两个分别有序的数组，时间复杂度为O(n)转换而来。 
步骤:依次将数组对半拆分，直到数组大小为1。将每个数组视为有序数组合并。
需要辅助数组 。

#### 堆排序
思路： 最大堆结构
步骤： 1.自下而上(下沉)建立大根堆，时间复杂度O(n)
       2.持续弹出(下沉)大根堆根节点，复用当前按最后一个叶子节点位置存储弹出的节点。不需要额外空间。 时间复杂度O(n log n)
	   PS： 如果用小根堆，需要额外空间进行存储。
####  计数排序
非比较排序
适用于 值类型不多(跨度很小)的数据。
思路：统计每个值类型的个数
     为了解决稳定性，计数数组的值不直接计算对应数值的个数和。而是计算小于等于当前数值的个数和(表示索引下标最大值)
#### 桶排序
**分治**的思想
**计数排序**和**归并排序**的结合
步骤：1.用自定义的映射函数(自定义)将数组映射到k个桶中 
       2.将桶中的元素进行排序 
       3.将桶进行合并 （归并排序）
最理想O(n+k) (k个桶中都一样大)，最差O(n^2)
优化：避免过多元素进入一个桶
	优化映射方法
	增加桶的数量，以空间换时间
	
#### 基数排序
**计数排序**一种应用
思路：对待排序元素的每一位依次进行**计数**排序。
步骤：先根据个位数大小排序，得到 [个位数最小的, 个位数第二小的... 个位数最大的]
	  再对十位数进行排序，
	  以此类推
	  借由计数排序是稳定的特性实现排序

## 3. 回溯
二叉树从底部到上遍历根据回溯。后序遍历就是回溯
	回溯的本质是**穷举**
- 回溯三要素：
	1.确认回溯函数模板的参数和返回值
	2.确认终止条件
	3.确认回溯搜索的遍历过程
#### 适用问题：
- 组合问题：N个数里面按一定规则找出k个数的集合 -> 找*叶子节点*
- 切割问题：一个字符串按一定规则有几种切割方式 -> 找*叶子节点*
- 子集问题：一个N个数的集合里有多少符合条件的子集 -> 找 *所有节点*
- 排列问题：N个数按一定规则全排列，有几种排列方式
- 棋盘问题：N皇后，解数独等等

## 容器适配器（container adapter）
底层不限制实现的容器，容器是可插拔的(即我们可以控制使用那种容器)
- 先进先出

### stack 栈
- 后进先出
- 无迭代器
- 底层数据结构: 数组或链表(可以指定 vector，deque，list)
	- SGI STL 默认使用deque 使用
- 入栈效率：O(1)。 如果是数组发生扩容时是O(n)
- 出栈效率:O(1)
### queue 队列
- 先进先出
- 底层数据结构: 可以指定 deque,list
		- SGI STL 默认使用deque 使用
- 入队效率：O(1)
- 出队效率: O(1)

### priority_queue  优先级队列
- 底层数据结构是大顶堆(完全二叉树)
- 插入效率:O(log n)
- 删除效率:O(log n)
### unordered_map
- 底层数据结构：哈希表（链地址法） 所有键哈希冲突，退化为链表（O(n)）或红黑树（O(log n)）
- 查找效率：平均O(1)，最差O(n)/O(log n) 。
### map
- 底层数据结构：红黑树（平衡二叉搜索树）
- 查找效率: 稳定O(log n[]())
