## 1.1. 寄存器
	以8086为例

### 1.1.1. 通用寄存器
用于存放一般性数据。16bit。
- AX（AH/AL）
- BX (BH/BL)
- CX (CH/CL)： 通常当作循环计数器
- DX (DH/DL)
- SI: 变址寄存器
- DI:变址寄存器
### 1.1.2. 段寄存器
	存放段地址
	不能将一个立即数传给段寄存器，必须通过一个寄存器将值传给段寄存器。
	 保护模式下，段寄存器存放的位段选择符，而非直接的地址

- CS： 代码段寄存器。存放指令(代码段)段地址 （CS:IP 指向下一条要读取的指令地址）
- DS:   数据段寄存器。存放段地址
- SS： 堆栈段寄存器。存放栈顶的段寄存器 (SS:SP 指向栈顶元素）
- ES:   扩展段寄存器。表示当前使用附加数据的段基地址。
- FS:  标志段寄存器。指向当前活动线程的TEB结构。80386开启增加的两个辅助段寄存器之一。
- GS: 全局段寄存器。80386开启增加的两个辅助段寄存器之一。

### 1.1.3. 专用寄存器
#### 1.1.3.1. 栈帧寄存器 SP

- SP 存放栈顶的偏移地址。 与SS堆栈段寄存器结合使用。
#### 1.1.3.2. 指针寄存器 IP
	IP 寄存器。 指令的偏移地址
	8086机：
		任意时刻，CPU将CS:IP（CS x 16 + IP）指向的内容当作指令执行。读取指令后，IP寄存器的值自动增加(增加的长度是对应指令的长度)。
		加电启动或复位后，CS设置为FFFFH，IP设置为0000H。因此FFFF0H时开机后执行的第一条指令。

#### 1.1.3.3. 标志寄存器 flag

具有一下三类作用
1. 用来存储相关指令的某些执行结果
2. 用来为CPU执行相关指令提供行为依据
3. 用来控制CPU的相关工作方式

其存储的信息通常被称为程序状态字（PSW）

|--|--|--|--|--|--|--|--|--|--|--|--|--|--|--|--|
|||||OF|DF|IF|TF|SF|ZF||AF||PF||CF|

##### 1.1.3.3.1. ZF 标志 
	 第6位。 
	 零标志位。
	 相关指令执行后，结果为0,ZF=1; 结果为1,ZF=0;

##### 1.1.3.3.2. PF 标志
	第2位。
	奇偶标志位。
	 相关指令执行后，所有bit位中1的个数位偶数,PF=1; 为奇数，PF=0

##### 1.1.3.3.3. SF 标志
	第7位。
	符号标志位。
	相关指令执行后，结果位负数,SF=1;非负数,SF=0。
 
##### 1.1.3.3.4. CF 标志
	第0位。
	进位标志位。
	一般在进行无符号数运算，记录运算结果的最高位有效位是否向更高位进位指，或从更高位的借位值。

##### 1.1.3.3.5. OF 标志位
	第11位。
	溢出标志位。 
	在进行有符号数运算时，发生溢出，OF=1; 没有溢出，OF=0

##### 1.1.3.3.6. DF标志位
	第10位。
	方向标志位。
	在串处理(movsb)指令中，控制每次操作后的si、di的增减

df = 0 每次操作后si、di递增
df = 1 每次操作后si、di递减

	本质上是修改CS和IP寄存器，以实现下一条指令的地址。

	无条件转移指令

	段间转移。远转移。
- jmp word ptr 内存单元地址

- jmp dword ptr  内存单元地址
	段间转移。内存单元地址高地址存放的是目的段地址。低地址存放的目的偏移地址。

# 5. 中断

CPU外部 法送过来或者内部产生的特殊信息，告知CPU不再接着（刚执行完的指令）向下执行，而转去处理这个特殊信息。

CPU 接受到中断信息后的步骤
1. （从中断信息中）取得中断类型码
2.   将标志寄存器中的值入栈
3.  将标志寄存器中的第8位TF和第9位IF设置为0 
4.  CS寄存器内容入栈
5.  IP寄存器内容入栈
6.  从内存地址为中断类型码*4 和中断类型码*4+2 的两个字单元读取中断处理函数的入口地址设置IP和CS 
## 5.1. 中断向量表
	映射中断码与对应中断处理函数的入口地址。
	8086PC放在内存地址 0000:0000到0000:03FF。的1024个单元中。一个表占两个字(4字节)。高地址字存放段地址，低地址字存放偏移地址

## 5.2. 中断处理函数

常规步骤：
1. 保存用到的寄存器
2. 处理中断
3. 恢复用到的寄存器
4. 用iret指令返回（iret 等效于 pop IP; pop CS; popf）

## 5.3. 中断类型

### 5.3.1. 内中断
#### 5.3.1.1. 除法错误
	中断码0
	比如dic指令发生除法溢出，
#### 5.3.1.2. 单步中断
	中断码1
	CPU执行完指令后。 如果检测导标志寄存器TF位为1，则产生中断过程。

#### 5.3.1.3. 执行info指令
	中断码4

#### 5.3.1.4. 执行int指令
	int n, 中断码即为n
	int 指令可以调用任一中断的中断处理函数

##### 5.3.1.4.1. int 13 

- ah: 函数表，0x02表示从驱动器读取扇区
	- dh: 磁头号
	- dl:驱动器号
	- dh:磁头号
	- ch:磁道号共有10位，低8位在dh表示
	- cl:位7、6表示磁道号高2位，位5-0表示起始扇区号(从1开始计)
	- al: 读取的扇区数
	- es:bx 表示要拷贝到的目的地址
### 5.3.2. 外中断


#### 5.3.2.1. 可屏蔽中断
	CPU可以不响应。当标志结存器IF位1时，则CPU在执行完当前指令后响应中断；若为0，则不响应可屏蔽中断。
	键盘输入的中断为中断类型码为9的可屏蔽中断。
#### 5.3.2.2. 不可屏蔽中断
	该类型中断固定码固固定为2. 所以中断过程中，不需要有取中断类型码的过程。
	CPU必须响应。






## 5.4. 响应中断的特殊情况

1.执行完ss寄存器传送数据的指令后，即使发生中断，CPU也不响应。
	为了保证SS和SP设置连续，否则中断过程压栈可能引起错误。
	（这能解释为什么使用Debug工具使用单步中断调试时执行设置SS后会连续执行设置SP）


# 6. 端口

指CPU通过总线连接的芯片。
包括 1. 接口卡(网卡、显卡)的接口芯片
	    2. 主板上的接口芯片，用于控制对外设访问
	    3. 其他芯片用于存储相关的系统信息或进行相关的输入输出处理。


## 6.1. CMOS RAM 芯片
1. 包含一个实时钟和一个128个存储单元的RAM存储器
2. 该芯片靠电池供电。因此关机仍可工作。

	0~0dh 保存时间，其余保存系统配置信息，供系统启动时BISO程序读取。

70h端口位地址端口。
71h端口为数据端口
