0.1. 三类指令#

  • 汇编指令： 机器码的助计符。
  • 伪指令： 没有对应机器码，由编译器执行，计算器不执行
  • 其他符号： 如 +、-、*、/ 等，由编译器识别，没有对应机器码

1. 存储单元

1Byte

1.1. 寄存器# 

以8086为例

1.1.1. 通用寄存器#

用于存放一般性数据。16bit。

  • AX（AH/AL）
  • BX (BH/BL)
  • CX (CH/CL)： 通常当作循环计数器
  • DX (DH/DL)
  • SI: 变址寄存器
  • DI:变址寄存器

1.1.2. 段寄存器# 

存放段地址
不能将一个立即数传给段寄存器，必须通过一个寄存器将值传给段寄存器。
 保护模式下，段寄存器存放的位段选择符，而非直接的地址
  • CS： 代码段寄存器。存放指令(代码段)段地址 （CS:IP 指向下一条要读取的指令地址）
  • DS: 数据段寄存器。存放段地址
  • SS： 堆栈段寄存器。存放栈顶的段寄存器 (SS:SP 指向栈顶元素）
  • ES: 扩展段寄存器。表示当前使用附加数据的段基地址。
  • FS: 标志段寄存器。指向当前活动线程的TEB结构。80386开启增加的两个辅助段寄存器之一。
  • GS: 全局段寄存器。80386开启增加的两个辅助段寄存器之一。

1.1.3. 专用寄存器#

1.1.3.1. 栈帧寄存器 SP#

  • SP 存放栈顶的偏移地址。 与SS堆栈段寄存器结合使用。

1.1.3.2. 指针寄存器 IP# 

IP 寄存器。 指令的偏移地址
8086机：
	任意时刻，CPU将CS:IP（CS x 16 + IP）指向的内容当作指令执行。读取指令后，IP寄存器的值自动增加(增加的长度是对应指令的长度)。
	加电启动或复位后，CS设置为FFFFH，IP设置为0000H。因此FFFF0H时开机后执行的第一条指令。

1.1.3.3. 标志寄存器 flag#

具有一下三类作用

  1. 用来存储相关指令的某些执行结果
  2. 用来为CPU执行相关指令提供行为依据
  3. 用来控制CPU的相关工作方式

其存储的信息通常被称为程序状态字（PSW）

5. 中断

CPU外部 法送过来或者内部产生的特殊信息，告知CPU不再接着（刚执行完的指令）向下执行，而转去处理这个特殊信息。

CPU 接受到中断信息后的步骤

  1. （从中断信息中）取得中断类型码
  2. 将标志寄存器中的值入栈
  3. 将标志寄存器中的第8位TF和第9位IF设置为0
  4. CS寄存器内容入栈
  5. IP寄存器内容入栈
  6. 从内存地址为中断类型码4 和中断类型码4+2 的两个字单元读取中断处理函数的入口地址设置IP和CS

5.1. 中断向量表# 

映射中断码与对应中断处理函数的入口地址。
8086PC放在内存地址 0000:0000到0000:03FF。的1024个单元中。一个表占两个字(4字节)。高地址字存放段地址，低地址字存放偏移地址

5.2. 中断处理函数#

常规步骤：

  1. 保存用到的寄存器
  2. 处理中断
  3. 恢复用到的寄存器
  4. 用iret指令返回（iret 等效于 pop IP; pop CS; popf）
