# Boot<no value>

	boot 使用的是as86汇编， 生成的是 16 位代码。 语法类似于 NASM、MASM

0. CPU自动进入实模式，并从0xFFFF0地址开始执行程序代码(ROM-BIOS) BIOS执行硬件检测诊断后，将物理地址处初始化为中断向量
1. boot加载
	1. boot程序大小512字节，放置在软盘映像文件的第一个扇区，
	2. ROM BIOS 先按启动盘启动顺序依次查询第一个扇区，如果第一个扇区最后两个字节(即第510、511字节)为0x55AA，则代表该启动盘可以启动
	则将启动盘第一个扇区加载到物理地址0x7c00(31k)位置，然后从0x7c00开始运行。 
		PS: 最早的个人PC芯片8088,搭配的操作系统是[86-DOS](https://en.wikipedia.org/wiki/86-DOS)。这个操作系统需要的内存最少是32KB(0x0000~0x7FFFF),8088芯片本身需要占用`0x0000～0x03FF`，用来保存[各种中断处理程序](https://en.wikipedia.org/wiki/BIOS_interrupt_call)的储存位置。（主引导记录本身就是中断信号INT 19h的处理程序。）所以，内存只剩下`0x0400～0x7FFF`可以使用。为了把尽量多的连续内存留给操作系统，主引导记录就被放到了内存地址的尾部。由于一个扇区是512字节，主引导记录本身也会产生数据，需要另外留出512字节保存。所以，它的预留位置就变成 `0x7FFF - 512 - 512 + 1 = 0x7C00`
	
1. boot 加载kernel
	1. boot 利用ROM BIOS 中断 int 0x13,将head代码读入到内存 0x10000(64KB)处（不直接读到内存0是因为BIOS的中断向量表在地址0处，内存1KB处是BIOS的数据区）
	2. 再将这段代码移动到内存0开输出
	3. 控制寄存器CR0，开启保护运行模式
	4. 跳到内存0处执行head代码
